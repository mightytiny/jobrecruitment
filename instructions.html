Page Specification

This document is a complete specification for building a professional bilingual recruitment sign-up page. Hand this to an AI coding assistant and ask it to build the page exactly as described.

1. Project Overview

Build a single-page recruitment website where job seekers and employers can register for free. The page must be bilingual — Khmer is the default language, with a toggle to switch to English. Design must be top professional.

What to build: One HTML page with two registration forms — one for job seekers, one for employers. When a user selects their type, the relevant form appears. On submit, data is saved to Firebase Firestore and any uploaded files go to Firebase Storage.

5. Language System

Every visible text string must exist in both Khmer and English using data-km and data-en HTML attributes. The lang.js file reads these attributes and swaps text when the user toggles language. The chosen language is saved in localStorage so it persists on refresh.

Default language is Khmer. The page loads in Khmer. The toggle button shows "EN" when in Khmer mode and "ខ្មែរ" when in English mode.

8. Firebase Logic (app.js)

Load Firebase SDK version 10.12.0 via CDN using ES module imports. Do not use npm.

Job Seeker registration flow

  1. Call createUserWithEmailAndPassword(auth, email, password)
  2. If a resume file was selected, upload it to Firebase Storage at path resumes/{userId}/{filename} and get the download URL
  3. Save all form data to Firestore collection seekers, document ID = user's UID
  4. Show success overlay. Reset form.
  5. On error, show error overlay with a friendly message (not the raw Firebase error code)

Employer registration flow

  1. Call createUserWithEmailAndPassword(auth, email, password)
  2. If a logo file was selected, upload it to Firebase Storage at path logos/{userId}/{filename} and get the download URL
  3. Save all form data to Firestore collection employers, document ID = user's UID. The "We Are Looking For" fields must be saved as a nested object called lookingFor
  4. Show success overlay. Reset form.
  5. On error, show error overlay with a friendly message

Common friendly error messages to handle

Firebase error codeMessage to show user
auth/email-already-in-useThis email is already registered. Please log in instead.
auth/invalid-emailPlease enter a valid email address.
auth/weak-passwordPassword must be at least 6 characters.
auth/network-request-failedNetwork error. Please check your internet connection.

9. Firebase Security Rules

Include these as comments at the bottom of app.js so the owner can copy and paste them into the Firebase console.

Firestore rules

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /seekers/{userId} {
      allow read:           if request.auth != null;
      allow create:         if request.auth != null && request.auth.uid == userId;
      allow update, delete: if request.auth.uid == userId;
    }
    match /employers/{userId} {
      allow read:           if request.auth != null;
      allow create:         if request.auth != null && request.auth.uid == userId;
      allow update, delete: if request.auth.uid == userId;
    }
  }
}

Storage rules

rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /resumes/{userId}/{fileName} {
      allow read:  if request.auth != null;
      allow write: if request.auth != null && request.auth.uid == userId;
    }
    match /logos/{userId}/{fileName} {
      allow read:  if true;
      allow write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

10. What the Owner Will Do After You Build It

  1. Create a free Firebase project at firebase.google.com
  2. Enable Authentication (Email/Password), Firestore, Storage, and Hosting
  3. Copy their Firebase config and paste it into the marked section of app.js
  4. Paste the security rules into Firebase console
  5. Drag the 4 files into Firebase Hosting to go live
